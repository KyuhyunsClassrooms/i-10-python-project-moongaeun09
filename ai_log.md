## AI 활용 성찰

Gemini가 도움이 된 점: 가구 배치 시 프로그램의 안정성을 높이기 위해 '방 범위 초과 검사'와 '가구 간 충돌 검사'를 어떤 논리적 흐름(이중 for문과 조건문의 결합)으로 짜야 하는지 알고리즘의 핵심 아이디어를 설계하는 데 큰 도움이 되었습니다. 또한 코드 수정 중 들여쓰기 오류(IndentationError)로 인해 수십 개의 에러가 발생했을 때, 당황하지 않고 구조가 정렬된 전체 코드를 통해 빠르게 디버깅할 수 있도록 가이드를 주어 문제 해결 시간을 단축해 주었습니다.


Gemini의 제안을 그대로 쓰지 않고 내가 판단하거나 수정한 점: AI가 처음 제안한 코드는 가구를 한 번 배치하면 프로그램이 종료되거나 배치 성공 여부만 간단히 확인하는 구조였습니다. 하지만 저는 실제 인테리어 레이아웃 시뮬레이터라는 프로그램 목적에 맞게, 사용자가 '종료'를 입력하기 전까지 가구를 계속해서 누적 배치할 수 있도록 메인 반복문(while True) 구조를 직접 설계하고 확장하여 적용했습니다. 

또한, 최종 실행 결과에서 모든 가구가 숫자 '1'로만 표현되어 서로 인접했을 때 구별하기 어렵다는 프로그램의 논리적 한계점을 AI의 제안에 의존하지 않고 스스로 테스트 환경(10x10 방 배정)을 통해 찾아내고 분석했습니다.


다음에 AI를 사용할 때 주의해야 할 점: AI가 제공하는 코드를 부분적으로 가져와 붙여넣을 때, 파이썬의 핵심 규칙인 들여쓰기(Indent)가 틀어지면 수십 개의 도미노 에러가 발생할 수 있다는 점을 배웠습니다. 다음에는 AI의 코드를 무조건 복사하기 전에 내 원래 코드의 문맥과 공백 규칙을 먼저 확인해야겠습니다. 

또한, AI가 제안한 알고리즘이 기술적으로는 오류 없이 작동하더라도, '실제 사용자가 쓸 때 편리한가(가구 구별이 안 되는 문제 등)'에 대한 논리적/시각적 한계는 AI가 대신 판단해 주지 않으므로, 항상 주도적인 비판적 시각을 가지고 코드를 검증해야겠다고 다짐했습니다.
