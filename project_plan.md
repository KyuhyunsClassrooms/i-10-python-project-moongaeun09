# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 가구 배치 도우미


---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.


내 설명: 방에 가구를 배치할 때는 단순히 전체 면적뿐만 아니라 가구의 가로 길이와 세로 길이도 고려해야 한다. 또한 가구의 방향을 바꾸어 배치하면 더 효율적으로 공간을 활용할 수 있다.
따라서 가구를 배치하기 전에 방 크기에 맞게 가구가 들어가는지, 회전했을 때 공간을 얼마나 차지하는지, 그리고 이미 배치한 다른 가구와 겹치지는 않는지 미리 시뮬레이션해 볼 수 있는 프로그램이 필요하다.

---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 방의 가로 크기(칸 수), 세로 크기(칸 수)

- 배치할 가구 종류 선택 (예: 1. 침대, 2. 책상, 3. 옷장 등)

- 가구 회전 여부 (Y/N)

- 가구를 배치할 시작 좌표 (X, Y)

- 프로그램 내부에 저장된 가구 데이터 (가구명, 가로 칸 수, 세로 칸 수)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 방 생성: 입력받은 방 크기대로 0으로 채워진 2차원 리스트(격자판)를 생성함.

- 회전 연산: 가구 회전 여부가 'Y'일 경우, 가구의 가로 길이와 세로 길이를 서로 맞바꿈.

- 범위 및 충돌 검사 (필터링): 가구가 방의 범위를 벗어나지 않는지 확인하고, 이중 반복문을 통해 놓으려는 자리에 이미 다른 가구가 배치되어 있는지(0이 아닌 숫자가 있는지) 검사함.

- 데이터 업데이트: 배치가 가능하면 2차원 리스트의 해당 범위를 가구 번호로 변경함.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 현재 방의 배치 상태를 시각적인 격자판(예: ⬜, 🛏️, 📝 등 모양이나 숫자)으로 출력함.

- 가구 배치 성공 메시지 또는 범위를 벗어나거나 겹쳤을 때의 예외 경고 메시지 출력.

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름: furniture_list (가구 도감 데이터) 및 room (방 배치도)


```python
# 
# furniture_list = [
```

### 한 행의 의미

프로그램에서 제공하는 가구 1개의 정보

### 각 열의 의미

| 열 번호 | 의미 | 예시 |
|---:|---|---|
| 0번 열 |가구 고유 번호 (ID)| 1 |
| 1번 열 |가구 이름|"침대"|
| 2번 열 |가구의 기본 가로 길이 (칸 수)|3|

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
|create_room|입력받은 크기의 빈 방(2차원 리스트) 생성|가로 크기, 세로 크기|0으로 채워진 2차원 리스트|
|display_room|현재 방의 가구 배치 상태를 화면에 출력|room (2차원 리스트)|없음 (화면 출력)|
|check_and_place|회전 반영, 범위 및 충돌 검사 후 가구 배치|room, 가구 가로/세로, 회전 여부, X/Y 좌표|배치 성공 여부 (True / False)|

## 6. 사용할 제어구조

### 조건문을 사용할 부분


내 계획: 사용자가 가구 회전(Y)을 선택했을 때 가로/세로 길이를 바꾸는 부분.

가구를 놓을 위치가 방의 크기를 벗어나는지(if start_x + f_w > room_w) 판단하는 부분.

가구를 놓을 자리에 이미 다른 가구의 값이 채워져 있는지(if room[y][x] != 0) 판단하는 부분.


### 반복문을 사용할 부분



내 계획: 방의 배치 상태(2차원 리스트)를 한 행씩 꺼내서 화면에 예쁘게 출력할 때 (for row in room).

가구가 차지할 공간에 다른 가구가 있는지 겹침 검사를 하거나, 실제 가구 값을 채워 넣을 때 사용하는 이중 for문.

사용자가 '종료'를 입력할 때까지 계속해서 가구 배치를 반복하는 while True 문.
