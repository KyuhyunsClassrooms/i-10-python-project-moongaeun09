# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 2차원 격자판 기반 가구 배치 도우미 (인테리어 레이아웃 시뮬레이터)

이 프로그램을 만든 이유: 방에 가구를 새로 배치할 때, 단순히 전체 면적만 계산해서는 가구의 실제 가로/세로 길이 때문에 배치가 불가능한 경우가 생긴다. 가구를 사전에 회전해 보거나 다른 가구와 겹치지 않는지 미리 시뮬레이션하여 공간을 효율적으로 활용할 수 있도록 돕기 위해 이 프로그램을 만들었다.


---

## 2. 문제 분해 및 모델링

### 입력 
 방의 가로/세로 크기, 배치할 가구 번호, 가구 회전 여부(Y/N), 가구를 놓을 시작 X/Y 좌표


### 처리
 1. 입력받은 방 크기만큼 0으로 채워진 2차원 격자판(방)을 생성한다.
2. 사용자가 가구를 회전하면 가로와 세로 길이를 스왑(Swap)한다.
3. 가구가 방 밖으로 나가는지(이탈) 검사하고, 이중 반복문으로 이미 다른 가구가 놓여 있는지(충돌) 검사한다.
4. 모든 검사를 통과하면 방 2차원 리스트의 값을 업데이트한다.


### 출력
 현재 가구가 배치된 방의 상태(2차원 리스트 형태), 배치 성공/실패 경고 메시지

---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: furniture_list (가구 도감 데이터)와 room (방 배치 격자판)

한 행의 의미: furniture_list에서 한 행은 가구 1개의 고유한 정보 정보

각 열의 의미: 

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 |가구 고유 번호 (ID) | 1 |
| 1번 열 |가구 이름 | "침대" |
| 2번 열 |가구의 가로 길이 (칸 수) | 3 |
| 3번 열 |가구의 세로 길이 (칸 수) | 2 |

이 2차원 리스트를 사용한 이유: 프로그램에 여러 가구의 이름과 크기 데이터를 체계적으로 저장하고, 사용자가 선택한 번호에 맞는 가구 정보를 반복문으로 간편하게 탐색하여 꺼내 쓰기 위해 사용했다. 또한 방 자체를 격자판으로 다루기 위해 0과 1로 구성된 2차원 리스트를 사용했다.


---

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
|create_room  | 지정한 크기대로 빈 방 생성 | 가로 크기(w), 세로 크기(h) | 0으로 채워진 2차원 리스트 |
| display_room | 현재 방의 배치 상태 출력 | room (2차원 리스트) | 없음 (화면 출력) |
| check_and_place | 가구의 회전, 이탈, 충돌을 검사하고 배치 | room, 가구 가로/세로 크기, 회전 여부, 시작 X/Y 좌표 | 배치 성공 여부 (True / False) |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름: check_and_place

설명: 이 함수는 사용자가 입력한 좌표에 가구를 안전하게 놓을 수 있는지 검증하는 예외 처리 시스템이다. 가구의 회전 조건문을 먼저 처리한 뒤, 방 크기를 벗어나는지 확인하고, 이중 for문을 돌려 해당 칸의 값이 0이 아닌지 검사하여 겹침(충돌) 현상을 완벽히 방어한다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?
if rotate == 'Y' or rotate == 'y': 문을 통해 가구 회전 여부를 검사했고, if (start_x + f_w > room_w) or... 문을 통해 가구가 방 밖으로 탈출하는 조건 및 이미 가구가 존재할 조건(if room[y][x] != 0)을 검사했다.

### 반복문 사용 부분

무엇을 반복 처리했는가?
 메인 루프인 while True 문을 통해 사용자가 '종료'를 입력하기 전까지 가구를 연속으로 배치할 수 있게 반복했다. 또한 for item in furniture_list:를 통해 도감을 순회하며 가구 정보를 탐색했다

### 조건문과 반복문이 함께 작동한 부분
 check_and_place 함수 내부의 이중 for문 안에서 if room[y][x] != 0: 조건문을 결합하여 가구가 놓일 모든 칸을 하나씩 확인하며 이미 가구가 있는지 실시간 충돌 검사를 수행했다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력: 방 크기: 가로 5, 세로 5

행동 선택: 추가

가구 번호: 1 (침대, 크기 3x2)

회전 여부: N

시작 좌표: X = 0, Y = 0

예상 결과: (0,0)부터 가로 3칸, 세로 2칸만큼 방 리스트의 값이 1로 채워지고 "✅ 침대 배치 성공!" 메시지가 출력된다.

실제 결과: 예상한 대로 (0,0) 좌표를 기준으로 가로 3바이트, 세로 2바이트 크기만큼 2차원 리스트 내부 데이터가 1로 업데이트되었으며 성공 메시지가 정상 출력되었다.


### 테스트 2: 예외 또는 경계 사례

입력: 이미 침대(1)가 배치된 (0, 0) 좌표에 다시 가구 '추가' 선택

가구 번호: 2 (책상, 크기 2x2)

회전 여부: N

시작 좌표: X = 0, Y = 0

예상 결과: 이미 침대가 자리를 차지하고 있으므로 충돌을 감지하여 ❌ [경고] 이미 그 자리에 다른 가구가 있어서 겹칩니다! 메시지를 출력하고 배치를 취소해야 한다.

실제 결과: 프로그램이 이중 for문을 통해 해당 좌표 공간을 조사하여 0이 아님을 감지했고, 예상했던 충돌 경고 메시지를 정확하게 출력하며 기존 가구 데이터를 안전하게 보호했다.

### 발견한 오류와 수정한 방법
 함수를 업그레이드하기 위해 코드를 부분적으로 개별 붙여넣는 과정에서, 파이썬의 구문 규칙인 들여쓰기가 어긋나면서 IndentationError를 비롯한 수십 개의 에러(문제 28개)가 한 번에 발생했다. 이를 해결하기 위해 위아래 코드의 구조와 괄호, 들여쓰기 공백(Tab) 규칙을 처음부터 다시 분석하였다. 결과적으로 개별 수정 대신 구조적 결함이 없는 완벽하게 정렬된 통합 코드를 재배치하는 방식을 활용하여 스스로 오류를 찾아내고 빌드를 성공시켰다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분: 아이디어를 구현하기 위한 2차원 리스트 구조 모델링과 충돌 및 이탈을 방지하는 예외 처리 로직의 논리적 흐름(이중 반복문 구조) 설계에 도움을 받았다.


Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분: 처음에는 가구를 1개만 배치하고 끝나는 간단한 함수를 제안받았으나, 인테리어 시뮬레이터라는 주제에 맞게 while 무한 루프를 적용하여 연속적으로 가구를 계속 누적 배치할 수 있도록 확장했다.


최종 코드에서 내가 직접 판단하거나 수정한 부분: 함수가 정상 작동하는지 확인하기 위해 직접 다양한 좌표와 크기를 대입해 보며 경계 조건 테스트를 수행했고, 꼬인 들여쓰기를 직접 검토하여 복구했다.

## 9. 한계와 개선 방안

현재 프로그램의 한계: 가구를 배치할 때 모든 가구가 똑같은 숫자 1로 저장되어, 나중에 어떤 가구가 침대고 어떤 가구가 책상인지 텍스트 화면만으로는 구별하기 어렵다


다시 개선한다면 추가하고 싶은 기능: 가구 도감의 고유 번호(침대는 1, 책상은 2)를 리스트에 직접 저장하고, display_room 함수에서 이 숫자를 식별하여 🛏️, 📝 같은 가구별 고유 이모지로 다르게 출력하는 시각화 기능을 개선하고 싶다.
